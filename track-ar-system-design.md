# 100m 赛道实时 AR 叠加系统 — 技术设计文档

## 1. 核心目标

8 条跑道、8 名运动员的 100m 直播画面中，在**不遮挡任何运动员**的前提下，为每一名运动员实时叠加：
- 当前名次（Rank）
- 当前成绩/时间（Time）
- 图形随运动员一起沿跑道前进

---

## 2. 零遮挡的核心原则

```
┌──────────────────────────────────────────────────────┐
│  每条跑道 = 约束走廊                                   │
│  跑道内部分为三个区域：                                  │
│                                                        │
│  [前方空域] ← 运动员 ← [后方空域]                     │
│                                                        │
│  → 图形固定放在 "前方空域"（运动员到终点之间）          │
│  → 或放在 "后方空域"（起点到运动员之间）               │
│  → 绝不覆盖运动员所在区域                               │
└──────────────────────────────────────────────────────┘
```

### 禁止性规则（硬约束）

| # | 规则 | 违反后果 |
|---|------|----------|
| 1 | 同一跑道内，图形 bounding box 不得与运动员 bounding box 在图像平面上有交集 | 直接遮挡 |
| 2 | 图形不得跨出本跑道边界 | 侵入邻道，可能遮挡邻道运动员 |
| 3 | 远侧跑道（透视下较小）的图形不应在图像空间上叠加到近侧跑道的运动员身上 | 透视遮挡 |

## 4. 相机与视角设计

### 推荐机位

```
         ┌─────┐
         │ CAM │  ← 侧上方 15-25m 高，俯角 25°-40°
         └──┬──┘
            │
    ──── 跑道 1 ────  (近端)
    ──── 跑道 2 ────
    ──── 跑道 3 ────
    ──── 跑道 4 ────
    ──── 跑道 5 ────
    ──── 跑道 6 ────
    ──── 跑道 7 ────
    ──── 跑道 8 ────  (远端)
    ================= 终点线

            ↑
         起跑线
```

**视角选择理由**：
- 侧上方俯角 → 每条跑道的视觉分离度最大，透视重叠最小
- 每条跑道在图像中形成近似平行的梯形区域
- 运动员沿 Y 方向运动，图形沿同一方向跟随
- 避免了正面视角下人侵人、道侵道的严重遮挡

### 备用机位

| 视角 | 优点 | 遮挡风险 | 推荐度 |
|------|------|----------|--------|
| 侧上方（推荐） | 跑道分离好，遮挡最小 | 低 | ⭐⭐⭐ |
| 正侧面（平视） | 运动员轮廓清晰 | 中——前后跑道的运动员可能在图像中重叠 | ⭐⭐ |
| 正上方（天眼） | 绝对无遮挡 | 无——但看不到运动员侧面，观赏性差 | ⭐⭐ |
| 终点正对 | 冲刺画面震撼 | 极高——8 人前后重叠 | ❌ |

## 5. 相机校准

### 5.1 已知几何先验

| 参数 | 值 |
|------|-----|
| 跑道长度 | 100.00 m |
| 跑道宽度 | 1.22 m |
| 跑道总数 | 8 |
| 跑道总宽 | 9.76 m |
| 内道内径 | 不计（直线段） |
| 起跑线位置 | X = 0 |
| 终点线位置 | X = 100 m |

### 5.2 坐标系定义

```
世界坐标系 (WCS):
  X: 沿跑道方向（起点→终点）
  Y: 垂直跑道方向（跑道 1→跑道 8）
  Z: 垂直地面向上
  原点: 跑道 1 与起跑线交点

像平面坐标系 (ICS):
  (u, v): 像素坐标

相机坐标系 (CCS):
  (Xc, Yc, Zc): 相机为中心
```

### 5.3 校准方式

使用 **PnP (Perspective-n-Point)** 求解相机位姿：

1. 开局时，操作员在画面上点选以下参考点：
   - 起点线与跑道 1、跑道 8 的交点（2 个点）
   - 终点线与跑道 1、跑道 8 的交点（2 个点）
   - 跑道 4 中点（可选，提升精度）

2. 已知这些点的世界坐标 → 求解相机外参 (R, t) 并假设内参已知（或使用预先标定）

3. **一次性校准**，比赛期间相机不允许移动（PTZ 需要动态重校准，暂不纳入第一版）

## 6. 运动员检测与跟踪

### 6.1 检测模型

| 组件 | 方案 |
|------|------|
| 模型 | YOLOv8n (Nano) 或 YOLOv8s (Small) |
| 类别 | person (COCO 预训练 + 田径场景微调) |
| 加速 | TensorRT / ONNX Runtime |
| 输入 | 1080p → 缩放到 640x640 |
| 输出 | 8 个 person bounding box |

### 6.2 身份分配策略（关键）

**核心思想：不用视觉 ReID，用几何约束做硬绑定。**

```
每帧得到 8 个 bbox:
  1. 对每个 bbox，取其底部中点 (u, v_bottom)
  2. 投影到跑道平面 → 得到世界坐标 (X, Y)
  3. 根据 Y 值判定属于哪个跑道: lane = floor((Y - Y_offset) / lane_width) + 1
  4. 同一跑道内，前后帧距离最近的 box 继承 ID
  5. 约束: 不允许 ID 跨跑道跳变 (即使检测失败也不接受跨道匹配)
```

**鲁棒性保证**：
- 卡尔曼滤波对每个跑道单独运行，预测当前帧运动员位置
- 检测丢失时，用卡尔曼预测值外推最多 5 帧
- 若同一跑道出现两个检测 → 选择与预测最接近的，报警提示人工介入

### 6.3 位置估计

得到运动员在世界坐标系中的 X 坐标（距起点距离）：

```
d_m = X  (从起跑线到运动员躯干中点的沿跑道距离)
```

同时记录：
- `frame_count`: 帧计数
- `timestamp`: 时间戳（用于计算成绩）
- `d_m`: 距离起跑线的米数

## 7. 排名与计时（无遮挡关键上下文）

### 7.1 实时排名

```
rank = sorted(athletes, key=lambda a: a.d_m, reverse=True)
```

按 `d_m` 从大到小排序，取索引为名次。
起跑瞬间所有运动员 `d_m ≈ 0`，排名按检测先后微调。

### 7.2 计时

有两种方案：

#### 方案 A：统一发令枪触发（推荐）

```
发令枪检测 → t_0 (所有运动员统一)
当前时间 t_i = frame_timestamp - t_0
成绩显示 = t_i (每位运动员相同，实时累加)
```

#### 方案 B：独立起跑检测

```
每位运动员的 t_0(lane) 由其离开起跑线（d_m > 0.3m）的帧决定
成绩显示 = t_i - t_0(lane)
```

方案 A 更容易保证公平性，也与正式比赛计时一致。

## 8. OCCLUSION GUARD — 核心安全机制

这是本系统的核心创新机制，保证**任何条件下图形不遮挡运动员**。

### 8.1 运动员占据区域建模

对于运动员 `i`，定义其在图像平面上的**占据区域**：

```
O_i = {
    bbox: [u_left, v_top, u_right, v_bottom],  // 检测框
    foot_pos: (u_foot, v_foot),                  // 脚底位置（投影到地面）
    head_pos: (u_head, v_head)                    // 头顶位置
}
```

### 8.2 图形安全锚点计算

图形放置在运动员**前方**（跑向终点方向），设定 3D 偏移量：

```
世界坐标系中:
  图形锚点 G_i_world = (X_i + offset_ahead, Y_lane_center, 0)

  其中:
    X_i = 运动员 i 的当前 X 坐标
    offset_ahead = 2.0 米 (可配置)
    Y_lane_center = 跑道 i 的中心线 Y 坐标

投影到图像:
  G_i_image = project(G_i_world)
```

### 8.3 安全碰撞检测（每帧执行）

```
for each lane i:
  1. 计算 G_i_image (图形锚点投影)
  2. 计算 O_i_image (运动员占据区域)
  3. 判断 G_i_image 是否在 O_i_image 内或边界重叠 < 10px:
     if collision:
       尝试以下策略 (按优先级)：
         a. 增大 offset_ahead (加大前方距离)
         b. 切换到 "后方" 放置: offset = -1.0m (图形放在运动员身后)
         c. 微调图形在垂直方向的偏移 (抬高或降低)
         d. 微调图形在水平方向的偏移 (靠跑道内侧或外侧)
     每次调整后重新检查碰撞，直到无碰撞
  4. 所有跑道检查完毕后，再做跨跑道交叉检查:
     for each (lane_i, lane_j):
       检查 G_i_image 是否与 O_j_image 在图像上重叠
       if cross_collision:
         微调 G_i 的水平位置 (靠向跑道内侧) 或降低高度
```

### 8.4 深度层级（Z-ordering）

渲染时按照距相机的深度排序：

```
深度 z 值（相机到物体的距离）:
  z(runner) < z(graphic_in_front)   // 运动员在图形前方 → 运动员覆盖图形

渲染顺序 (从远到近):
  1. 跑道地面
  2. 远端跑道图形
  3. 远端跑道运动员
  4. 近端跑道图形
  5. 近端跑道运动员

最终效果: 运动员始终在图形之上
          (即使图形投影在运动员区域附近，深度测试会保证运动员优先显示)
```

### 8.5 安全边界冗余

```
┌───────────────────── 跑道边界 ─────────────────────┐
│                                                      │
│   [图形安全区]   ← 2.0m →  [运动员]  ← 1.0m →  [空] │
│                                                      │
│   │← 跑道宽度 1.22m →│                                 │
└──────────────────────────────────────────────────────┘
```

- 图形与运动员之间保留 **至少 0.5m（3D 世界空间）** 的物理间隔
- 图形高度限制在跑道平面的 **±0.3m（世界空间）** 以内，避免浮入空中遮挡远处运动员
- 图形水平宽度不超过 **0.8 * 跑道宽度**，保留两侧安全边距

## 9. 图形渲染设计

### 9.1 图形内容

```
┌──────────────────┐
│ ①  #3            │  ← 名次 (大号数字)
│  08.45s          │  ← 时间成绩
│  [RUNNER NAME]   │  ← 运动员名称（可选）
└──────────────────┘
```

### 9.2 3D 空间布局

- 图形永远平贴在地面平面上（Z=0），作为「地面贴花」
- 或者略微抬高（Z=0.1m）以防止被草皮纹理遮盖，但仍在地面附近
- 图形始终朝向相机（billboard 模式）以保证可读性

### 9.3 地面贴花方案（推荐）

```
将图形渲染为跑道地面上的 textured quad:
  - 四边形贴合跑道平面
  - 跟随运动员前进 (offset_ahead)
  - 透视畸变自然由投影处理
  
优点:
  - 深度测试自动处理 → 运动员跨过图形时自然覆盖图形
  - 不会因为浮在空中而遮挡远处跑道上的运动员
  - 视觉上融合度好，观众感觉是"印在跑道上的"
```

## 10. 极端情况处理

| 场景 | 风险 | 应对 |
|------|------|------|
| **运动员摔倒** | 其他运动员赶上，图形覆盖倒地者 | 检测到 bbox 高度异常（< 正常 50%）或速度骤降 → 该跑道图形向后偏移至身后区域 |
| **终点冲刺（8 人并排）** | offset_ahead 空间不足（距终点 < 2m） | 自动切换为 offset_behind = -1.5m，图形放在运动员身后 |
| **运动员大幅领先（拉开 > 5m）** | 前方空域巨大，图形距运动员过远 | offset_ahead 动态自适应：`offset = clamp(distance_to_next * 0.3, 1.0, 5.0)` |
| **运动员串道** | ID 冲突、遮挡异常 | 检测到跨道行为 → 暂时将该运动员的图形透明度调低并显示警告标记 |
| **极端天气/低光照** | 检测精度下降 | 启用 IR 模式或低光增强预处理；检测置信度 < 阈值时 fallback 到卡尔曼预测 |
| **相机抖动** | 图形位置跳动 | 卡尔曼平滑 + 图形位置使用 3 帧指数移动平均 |

for each frame in test_sequence:

        bbox = athlete_bbox

## 16. 附录：Occlusion Guard 伪代码

```
function computeSafePosition(athlete, lane_geometry, camera):
    // 1. 计算默认前方位置
    offset_ahead = 2.0  // 米
    G_world = (athlete.x + offset_ahead, lane_geometry.center_y, 0)
    
    // 2. 投影到图像
    G_img = project(G_world, camera)
    athlete_bbox = athlete.detection_bbox
    
    // 3. 碰撞检测
    while rects_overlap(G_img.bbox, athlete_bbox):
        // 尝试增大前方距离
        offset_ahead += 0.5
        if offset_ahead > 5.0:
            // 前方空间不足，切换到后方
            offset_behind = 1.0
            G_world = (athlete.x - offset_behind, lane_geometry.center_y, 0)
            G_img = project(G_world, camera)
            if rects_overlap(G_img.bbox, athlete_bbox):
                offset_behind += 0.5  // 继续拉远
            
            // 如果后方也撞 (不可能，除非检测错误)
            if offset_behind > 3.0:
                G_world = (athlete.x, lane_geometry.center_y - 0.4, 0.3)
                // 极限 fallback: 在跑道左侧偏低位置，抬高悬浮
        
        G_world = (athlete.x + offset_ahead, lane_geometry.center_y, 0)
        G_img = project(G_world, camera)
    
    // 4. 跨跑道碰撞检查
    for other_athlete in all_athletes:
        if other_athlete.lane != athlete.lane:
            other_bbox = other_athlete.detection_bbox
            if rects_overlap(G_img.bbox, other_bbox):
                // 水平微移向跑道内侧
                G_world.y -= 0.15
                G_img = project(G_world, camera)
                if rects_overlap(G_img.bbox, other_bbox):
                    G_world.y += 0.3  // 试另一侧
    
    return G_world, G_img
```
